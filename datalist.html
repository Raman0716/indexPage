  • 建模目标

    1. 预测给定地理区域在未来某个时间段的出行需求量及需求满足量。

    2. 调研发现，同一地区不同时间段的订单密度是不一样的，例如大型居住区在早高峰时段的出行需求比较旺盛，而商务区则在晚高峰时段的出行需求比较旺盛。如果能预测到在未来的一段时间内某些地区的出行需求量比较大，就可以提前对营运车辆提供一些引导，指向性地提高部分地区的运力，从而提升乘客的整体出行体验。

  • 字段说明

    订单信息表

    字段名 类型 含义 示例
    order_id String 订单号 12345678
    date String 日期 2017/01/01

    订单信息表主要覆盖了一张订单的基本信息，包括这张订单的日期，以及价格（NULL表示为空）。

  • 评价指标

    对n个区域和q个时间片，区域di在时间片tj的供需缺口为gapij，选手预测值为sij，以MAE作为最终的评价指标：MAE越小越好。

  • 组队规则

    1. 团队人数上限5人

    2. 在第一阶段的最后3天无法新建队伍，但是可以加入其它队伍。

    3. 在最后一个阶段最后3天无法新建队伍，无法加入队伍。

    4. 竞赛进入历史阶段后解除一切限制，注意：答辩队伍成员仅限活跃期间加入的成员。（引用DataCastle)
